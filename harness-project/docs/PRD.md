# PRD: CS Quiz

## 목표
CS 기초 지식을 객관식 문제 풀이로 반복 학습할 수 있는 웹 사이트를 만든다.

## 사용자
CS 지식을 점검하거나 학습하고 싶은 개발자 지망생 및 현직 개발자.

---

## 핵심 기능

### 1. 시작 화면 (`/`)
- 사이트 타이틀 + 한 줄 소개
- "문제 풀기 시작" 버튼 → `/quiz`로 이동
- 전체 문제 수, 출제 문제 수(30개) 표시

### 2. 퀴즈 진행 화면 (`/quiz`)

#### 초기화
- 진입 시 전체 문제 풀(pool)에서 30개를 랜덤 샘플링하여 퀴즈 세션 구성
- 세션은 컴포넌트 마운트 시 1회만 초기화된다

#### 문제 표시
- 문제는 1번부터 30번까지 순차 표시
- 각 문제: 카테고리 배지 + 질문 텍스트 + 4지선다 보기 (A/B/C/D 레이블)
- 현재 문제 번호 및 진행률 표시 (예: `7 / 30`)
- 진행률 바 (선택 완료된 문제 수 / 30 기준)
- **보기 순서**: `questions.ts`에 정의된 순서 그대로 표시. 보기 셔플 없음.
- **출제 순서**: 30문제는 카테고리 정렬 없이 샘플링된 랜덤 순서 그대로 출제

#### 답안 선택
- 보기 클릭 시 해당 보기가 선택 상태로 강조
- 이미 선택한 보기를 다시 클릭해도 선택 해제되지 않는다 (한 번 선택하면 반드시 다른 보기를 클릭해야 변경)
- 선택 변경은 제출 전까지 자유롭게 가능
- **보기 선택 후 자동 이동 없음**: 보기를 선택해도 다음 문제로 자동 이동하지 않는다. 사용자가 직접 "다음" 버튼 또는 네비게이터를 눌러야 한다 (검토/변경 기회 보장)

#### 네비게이션
- 이전 / 다음 버튼으로 문제 이동
- 첫 번째 문제에서 "이전" 버튼 비활성화
- 마지막 문제에서 "다음" 버튼 비활성화
- 문제 번호 점프 네비게이터: 1~30 번호 버튼 표시
  - 미선택: 기본 스타일
  - 선택 완료: 완료 스타일 (색상 구분)
  - 현재 문제: 강조 스타일

#### 제출
- "제출" 버튼은 30문제 모두 선택 완료 시에만 활성화 (미완료 시 `disabled`)
- 제출 버튼 위에 미선택 문제 수 표시 (예: `3문제 미선택`)
- 제출 클릭 시 확인 모달 없이 즉시 채점 후 `/result`로 이동

### 3. 결과 화면 (`/result`)

#### 요약
- 총점: `N / 30`
- 정답 수, 오답 수 표시
- 점수에 따른 간단한 평가 메시지 (예: 90% 이상 "우수", 70% 이상 "양호", 미만 "분발")

#### 오답 해설
- 틀린 문제만 목록으로 표시 (맞힌 문제는 표시 안 함)
- 오답 항목은 접기/펼치기 없이 전체 표시 (MVP에서 accordion UI 없음)
- 각 오답 항목:
  - 문제 번호 + 카테고리 배지
  - 문제 텍스트
  - 보기 4개 전부 표시 (내 오답 → 빨간색, 정답 → 초록색, 나머지 → 흐리게)
  - 간단한 해설 (1~3문장)
- 모두 맞혔을 경우: "모든 문제를 맞혔습니다!" 메시지 표시, 오답 목록 없음

#### 재시작
- "다시 풀기" 버튼 → `/quiz`로 이동하여 새로운 랜덤 30문제로 재시작
- "홈으로" 버튼 → `/`로 이동

## 에러 케이스 및 엣지 케이스

### E-1. `/result` 직접 접근 (데이터 없음)
- sessionStorage에 QuizResult 데이터가 없을 때 `/result`에 직접 접근한 경우
- **동작**: 즉시 `/`로 리다이렉트

### E-2. sessionStorage 사용 불가
- 일부 브라우저 private 모드에서 sessionStorage 쓰기가 차단될 수 있음
- **동작**: try/catch로 감지 → 에러 시 `/`로 리다이렉트 + 콘솔 경고

### E-3. 페이지 새로고침 (퀴즈 진행 중)
- `/quiz`에서 새로고침 시 useState 기반 퀴즈 상태가 초기화됨
- **동작**: 퀴즈를 새로 시작한다 (문제가 새로 샘플링됨). 별도 안내 없음.
- **의도적 결정**: 진행 상태를 localStorage/sessionStorage에 저장하지 않는다 (MVP 범위 외)

### E-4. 문제 풀(pool)이 30개 미만
- `questions.ts`의 문제 수가 30개 미만인 경우 (데이터 관리 실수)
- **동작**: `sample.ts`에서 guard 처리 — pool 전체를 반환하고 콘솔 에러 출력
- **방지**: CI에서 문제 수 30개 이상 검증하는 테스트를 포함

### E-5. 브라우저 뒤로가기 (`/result` → `/quiz`)
- 결과 화면에서 뒤로가기 시 퀴즈 화면으로 돌아갈 수 있음
- **동작**: `/quiz` 재진입 시 새로운 퀴즈 세션으로 초기화됨 (이전 답안 복원 없음)

### E-6. 0문제 선택 후 제출 시도
- "제출" 버튼은 30문제 전부 선택 완료 시에만 활성화되므로 이 케이스는 UI 레벨에서 차단됨
- disabled 상태의 버튼은 클릭 이벤트를 발생시키지 않는다

### 5. 퀴즈 선택 화면 (`/quiz`)

기존 `/quiz`(진행 화면)는 `/quiz/play`로 이동. `/quiz`는 선택 화면이 된다.

- **ALL**: official + approved 문제 전체에서 랜덤 30개
- **카테고리 6개**: 선택한 카테고리에서 랜덤 30개
  - 문제 수 표시: "자료구조 (42문제)"
  - 10개 미만 시 비활성화 + "최소 10개 필요"
  - 로그인 사용자: 내 카테고리 정답률 표시

---

### 6. 게시판 (`/board`, `/board/[id]`, `/board/submit`)

#### 목록 (`/board`)

- **키워드 검색**: URL `?q=` 파라미터, 서버사이드 ILIKE 검색 (문제 텍스트 대상)
- **필터**: 카테고리 (전체 + 6개), 상태 (전체 / 등록요청 / 승인)
- **정렬**: 등록일 최신순(기본) / 정답률 낮은순 / 정답률 높은순 / 좋아요순
- **카드**: 닉네임, 카테고리·상태 뱃지, 시도횟수, 정답률, 좋아요수, 등록일, 문제 앞 50자
- **페이지네이션**: 20개/페이지 (페이지 기반)
- `BLINDED` 문제: 일반 사용자에게 숨김
- 비로그인: 목록/상세 허용

#### 상세 (`/board/[id]`)

- 문제 전체 (보기 4개, 정답, 해설)
- 작성자 닉네임, 등록일, 상태 뱃지
- 통계: 시도 횟수 N회, 정답률 N%
- **좋아요** 버튼 (로그인 시 토글. 비로그인 클릭 → 로그인 유도)
- **신고** 버튼 (로그인 + 본인 제외):
  - 이유: 부적절한 내용 / 오류 있음 / 중복 문제 / 기타
  - 부가 설명 (선택, 최대 200자)
  - 이미 신고한 문제: "신고 완료" 상태, 재신고 불가
- 직접 풀기 없음 (퀴즈에서만 풀기)

#### 문제 등록 (`/board/submit`)

- 로그인 + 닉네임 필수
- 필드: 카테고리, 문제(최대 500자), 보기 A~D(각 200자), 정답 선택, 해설(최대 500자)
- 제출 → `status: PENDING` 저장, 게시판에 즉시 노출 → `/board` 이동

---

### 7. 관리자 패널 (`/admin`)

- `role: ADMIN` 전용. 미들웨어에서 일반 사용자 차단 → `/` 리다이렉트

**탭 1: 승인 대기**
- PENDING 문제 목록 (등록일순)
- 승인 → `status: APPROVED` + `QUESTION_APPROVED` 알림 생성
- 거절 → 이유 입력 모달 → `status: REJECTED` + `rejectionReason` 저장 + `QUESTION_REJECTED` 알림 생성

**탭 2: 신고 접수**
- PENDING 신고 목록 (신고 수 많은 순)
- 블라인드 → `status: BLINDED` (퀴즈·게시판에서 제외)
- 무시 → `report.status: REVIEWED`

---

### 8. 메인 랭킹 (`/`)

- 기존 시작 화면 유지 + 하단에 카테고리별 TOP 5 랭킹 섹션 추가
- 랭킹 기준: 정답률 (동률 → 시도 횟수 많은 순)
- 진입 조건: 해당 카테고리 10문제 이상 시도
- 컬럼: 순위, 닉네임, 시도 횟수, 정답률
- 로그인 사용자: 본인 행 하이라이트
- 캐시: 60초 revalidation

---

### 9. 마이페이지 확장 (`/mypage`)

기존 기능(풀이 히스토리, 오답노트, 카테고리 통계)은 localStorage → DB 기반으로 전환.

추가 기능:
- **요약 카드**: 총 퀴즈 횟수, 전체 정답률, 연속 기록(스트릭 N일)
- **내가 등록한 문제** 탭: 상태별 필터. 거절된 문제: 거절 이유 + 수정 후 재요청 버튼
- **내가 좋아요한 문제** 탭: 게시판 상세 링크

### 10. 알림

- 헤더 벨 아이콘: 미읽은 수 배지 (로그인 시)
- 드롭다운 목록:

- "모두 읽음" 버튼

## 비기능 요구사항 (v2 추가)

- **보안**: 모든 POST/PATCH API는 서버 세션 검증 필수. 관리자 API는 `role: ADMIN` 추가 검증.
- **DB 성능**: 퀴즈 제출 시 Prisma `$transaction`으로 question_attempts + question 통계 컬럼 원자적 업데이트.
- **검색 성능**: `questions.question` 컬럼 ILIKE. 데이터 증가 시 `pg_trgm` GIN 인덱스 적용 권장.
- **랭킹 캐싱**: 60초 `next.revalidate`.

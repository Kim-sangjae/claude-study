# PRD: CS Quiz

## 목표
CS 기초 지식을 객관식 문제 풀이로 반복 학습할 수 있는 웹 사이트를 만든다.

## 사용자
CS 지식을 점검하거나 학습하고 싶은 개발자 지망생 및 현직 개발자.

---

## 핵심 기능

### 1. 시작 화면 (`/`)
- 사이트 타이틀 + 한 줄 소개
- "문제 풀기 시작" 버튼 → `/quiz`로 이동
- 전체 문제 수, 출제 문제 수(30개) 표시

### 2. 퀴즈 진행 화면 (`/quiz`)

#### 초기화
- 진입 시 전체 문제 풀(pool)에서 30개를 랜덤 샘플링하여 퀴즈 세션 구성
- 세션은 컴포넌트 마운트 시 1회만 초기화된다

#### 문제 표시
- 문제는 1번부터 30번까지 순차 표시
- 각 문제: 카테고리 배지 + 질문 텍스트 + 4지선다 보기 (A/B/C/D 레이블)
- 현재 문제 번호 및 진행률 표시 (예: `7 / 30`)
- 진행률 바 (선택 완료된 문제 수 / 30 기준)
- **보기 순서**: `questions.ts`에 정의된 순서 그대로 표시. 보기 셔플 없음.
- **출제 순서**: 30문제는 카테고리 정렬 없이 샘플링된 랜덤 순서 그대로 출제

#### 답안 선택
- 보기 클릭 시 해당 보기가 선택 상태로 강조
- 이미 선택한 보기를 다시 클릭해도 선택 해제되지 않는다 (한 번 선택하면 반드시 다른 보기를 클릭해야 변경)
- 선택 변경은 제출 전까지 자유롭게 가능
- **보기 선택 후 자동 이동 없음**: 보기를 선택해도 다음 문제로 자동 이동하지 않는다. 사용자가 직접 "다음" 버튼 또는 네비게이터를 눌러야 한다 (검토/변경 기회 보장)

#### 네비게이션
- 이전 / 다음 버튼으로 문제 이동
- 첫 번째 문제에서 "이전" 버튼 비활성화
- 마지막 문제에서 "다음" 버튼 비활성화
- 문제 번호 점프 네비게이터: 1~30 번호 버튼 표시
  - 미선택: 기본 스타일
  - 선택 완료: 완료 스타일 (색상 구분)
  - 현재 문제: 강조 스타일

#### 제출
- "제출" 버튼은 30문제 모두 선택 완료 시에만 활성화 (미완료 시 `disabled`)
- 제출 버튼 위에 미선택 문제 수 표시 (예: `3문제 미선택`)
- 제출 클릭 시 확인 모달 없이 즉시 채점 후 `/result`로 이동

### 3. 결과 화면 (`/result`)

#### 요약
- 총점: `N / 30`
- 정답 수, 오답 수 표시
- 점수에 따른 간단한 평가 메시지 (예: 90% 이상 "우수", 70% 이상 "양호", 미만 "분발")

#### 오답 해설
- 틀린 문제만 목록으로 표시 (맞힌 문제는 표시 안 함)
- 오답 항목은 접기/펼치기 없이 전체 표시 (MVP에서 accordion UI 없음)
- 각 오답 항목:
  - 문제 번호 + 카테고리 배지
  - 문제 텍스트
  - 보기 4개 전부 표시 (내 오답 → 빨간색, 정답 → 초록색, 나머지 → 흐리게)
  - 간단한 해설 (1~3문장)
- 모두 맞혔을 경우: "모든 문제를 맞혔습니다!" 메시지 표시, 오답 목록 없음

#### 재시작
- "다시 풀기" 버튼 → `/quiz`로 이동하여 새로운 랜덤 30문제로 재시작
- "홈으로" 버튼 → `/`로 이동

## 에러 케이스 및 엣지 케이스

### E-1. `/result` 직접 접근 (데이터 없음)
- sessionStorage에 QuizResult 데이터가 없을 때 `/result`에 직접 접근한 경우
- **동작**: 즉시 `/`로 리다이렉트

### E-2. sessionStorage 사용 불가
- 일부 브라우저 private 모드에서 sessionStorage 쓰기가 차단될 수 있음
- **동작**: try/catch로 감지 → 에러 시 `/`로 리다이렉트 + 콘솔 경고

### E-3. 페이지 새로고침 (퀴즈 진행 중)
- `/quiz`에서 새로고침 시 useState 기반 퀴즈 상태가 초기화됨
- **동작**: 퀴즈를 새로 시작한다 (문제가 새로 샘플링됨). 별도 안내 없음.
- **의도적 결정**: 진행 상태를 localStorage/sessionStorage에 저장하지 않는다 (MVP 범위 외)

### E-4. 문제 풀(pool)이 30개 미만
- `questions.ts`의 문제 수가 30개 미만인 경우 (데이터 관리 실수)
- **동작**: `sample.ts`에서 guard 처리 — pool 전체를 반환하고 콘솔 에러 출력
- **방지**: CI에서 문제 수 30개 이상 검증하는 테스트를 포함

### E-5. 브라우저 뒤로가기 (`/result` → `/quiz`)
- 결과 화면에서 뒤로가기 시 퀴즈 화면으로 돌아갈 수 있음
- **동작**: `/quiz` 재진입 시 새로운 퀴즈 세션으로 초기화됨 (이전 답안 복원 없음)

### E-6. 0문제 선택 후 제출 시도
- "제출" 버튼은 30문제 전부 선택 완료 시에만 활성화되므로 이 케이스는 UI 레벨에서 차단됨
- disabled 상태의 버튼은 클릭 이벤트를 발생시키지 않는다

## 문제 데이터 범위 및 품질 기준

아래 CS 주제에서 골고루 출제한다 (최소 120문제 확보, 카테고리당 최소 15문제):

| 카테고리 | 코드 | 최소 문제 수 |
|---------|------|------------|
| 자료구조 | `ds` | 20 |
| 알고리즘 | `algo` | 20 |
| 운영체제 | `os` | 20 |
| 네트워크 | `network` | 20 |
| 데이터베이스 | `db` | 20 |
| 컴퓨터 구조 | `arch` | 20 |

**문제 품질 기준:**
- 보기 4개가 모두 그럴듯해야 한다 (trivially 틀린 보기 금지)
- 해설은 왜 정답인지 + 왜 오답인지 핵심만 1~3문장으로
- 중복 문제 금지 (동일 개념의 변형 문제는 허용)

## 비기능 요구사항

- **반응형**: 모바일(375px~) / 태블릿 / 데스크탑 모두 사용 가능해야 함
- **성능**: 초기 로딩 후 문제 전환은 클라이언트 사이드 — 추가 네트워크 요청 없음
- **접근성**: 보기 버튼은 키보드(Tab + Enter/Space)로 선택 가능해야 함
- **브라우저 탭 타이틀**: 전체 공통 `"CS Quiz"` (페이지별 suffix 없음)
- **404 페이지**: 존재하지 않는 경로 접근 시 Next.js 기본 404 사용 (커스텀 불필요)

---

## MVP 제외 사항
- 사용자 계정 / 로그인
- 점수 히스토리 저장 (localStorage 포함)
- 문제 카테고리 필터링
- 문제 난이도 분류
- 관리자 문제 편집 UI
- 시간 제한 기능
- 퀴즈 진행 상태 저장 (새로고침 시 복원)
